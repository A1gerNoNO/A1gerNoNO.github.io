## 工程结构是什么
```
哪里写业务逻辑
哪里访问数据库
哪里调 Redis
哪里调外部接口
哪里接收 HTTP / MQ / 定时任务
哪里放通用异常、日志、配置
```
## 推荐分层
```
app
api
trigger
domain
infrastructure
types
case / application
```
## DDD常见模块划分
```
app              应用启动、配置、AOP、环境配置
api              对外接口契约
trigger          HTTP、RPC、MQ、定时任务等入口
domain           领域模型和核心业务规则
	model
		aggregate   聚合
        entity      实体
        valobj      值对象
    repository
    service
infrastructure   数据库、Redis、外部接口、MQ 等技术实现
types            通用对象、枚举、响应体、异常码
```
## 一次请求的调用链路
```
HTTP 请求
↓
trigger 接收请求
↓
application / case 编排流程
↓
domain 执行业务规则
↓
repository 定义数据访问需求
↓
infrastructure 访问数据库、缓存或外部系统
↓
返回结果
```
